# Dynamic room scaling and camera follow (v2)

The living room environment now dynamically resizes to enclose tracks of any extent, and the camera smoothly follows the car during play mode. All three issues from the v1 review have been addressed: `followCar` correctly swaps Y/Z axes, `_rebuildEnvironmentForTrack` guards against redundant rebuilds via `_currentRoomHalf`, and the shadow frustum scales with room size.

**Watch for:** Wall height may be exceeded by tall looping tracks (possible) — only horizontal extent is measured, so a track with large vertical loops could poke above the walls. Environment position isn't updated after a rebuild triggered by a size change (likely) — currently safe because the only caller is `_recenterCamera` which sets position afterward, but fragile if `_rebuildEnvironmentForTrack` is ever called from a different site.

**Verdict**: APPROVED

## High-level view

The room-scaling mechanism computes the track's XZ bounding box from all piece entry states, derives a square `roomHalf` (clamped to minimum 16, padded by 8 units), and fully disposes/rebuilds the environment when the size changes. The `_currentRoomHalf` guard eliminates redundant rebuilds on mode switches. Wall height, fog range, shadow frustum, and floor size all scale proportionally.

The camera follow uses exponential interpolation with the correct Y/Z swap, and the reset path on mode switch restores the overview via `_recenterCamera`. Both v1 concerns in this area are resolved.

## Vertical extent ignored in room sizing

The bounding box calculation only measures `gx` and `gy` (horizontal position). The `gz` component (height in grid space, Three.js Y) is unused. Wall height scales as `roomHalf * (9/16)`, which for the default room of 16 gives 9 units. The risk is a narrow-but-tall track where roomHalf stays at 16 but the track reaches 12+ units high — the track would poke above the walls.

```typescript
const wallHeight = Math.max(9, roomHalf * (9 / 16));
```

A wide track (roomHalf=30) gets walls of ~17 units, which is generous. The edge case is a compact track with stacked loops. **Confidence: possible** — no standard piece sequence was verified to produce this scenario.
